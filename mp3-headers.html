Avoiding MP3 Frame Headers

If you have ever tried to databend an MP3 file in a text editor you might have been disappointed by the results â€” which is often just an unplayable file. MP3 files organize data into a series of "frames" each starting with their own header, which means the headers are distributed throughout the file and thus the chances of corrupting it beyond interpretation are much higher. This page allows you to upload (or drag and drop) an MP3 file to analyze it's data and identify what it's frame header looks like. You can use this information to avoid hacking the frame headers when databending an Mp3 file in your own hex editor. You can also use the "find and replace" tool below which avoids manipulating any of the bytes in the frame headers, resulting in an always playable glitched MP3.
